 ‚úÖ Q2: var, let aur const mein kya farq hota hai?

 var -> function scoped hota hai
 let  -> block scoped hota hai
 const -> block scoped hota hai

 var ko dobara declare aur assign dono kiya ja sakta hai
 let ko dobara assign kar sakte hain, lekin dobara declare nahi kar sakte
 const ko na dobara assign kar sakte hain, na hi declare

 ‚úÖ Example:
if (true) {
    var x = 10;
    let y = 20;
    const z = 30;
}

console.log(x);  ‚úÖ chalega (var function scoped hota hai)
 console.log(y);  ‚ùå error (let block ke bahar accessible nahi hota)
 console.log(z);  ‚ùå error (const bhi block ke bahar nahi chalega)



 ‚úÖ Q3: Login ke case mein let aur const ka use kaise karte hain?

 const ka use fixed values ke liye hota hai jaise: VALID_EMAIL, API_URL
 let ka use hota hai changing values ke liye jaise: isLoggedIn, userInput

    const VALID_EMAIL = "aman@example.com";

 ‚úÖ Q5: Kya let aur const hoist hote hain?

 Haan, hoist hote hain lekin Temporal Dead Zone (TDZ) mein hote hain
 iska matlab yeh ke declaration se pehle unko access nahi kar sakte

 console.log(a);  ‚ùå error (TDZ issue)
let a = 10;

 console.log(b);  ‚ùå error
const b = 20;


 ‚úÖ Summary:
 - let use karo jab value change hone wali ho
 - const use karo jab value fix ho
 - var ko avoid karo modern JavaScript mein

6. JavaScript mein kitne types ke data types hote hain? Unka naam bataen.

Answer:
JavaScript mein 8 data types hote hain:

Primitive Data Types:

* String
* Number
* Boolean
* Null
* Undefined
* Symbol (ES6)
* BigInt (ES11)

Non-Primitive Data Type:

* Object



7. Primitive aur Non-Primitive data types mein kya farq hota hai?

Answer:

Primitive:

* Value directly store hoti hai
* Immutable hote hain
* Compare by value
* Examples: number, string, boolean

Non-Primitive (Reference):

* Reference store hota hai
* Mutable hote hain
* Compare by reference
* Examples: object, array, function



8. JavaScript mein null aur undefined mein kya difference hai?

Answer:

undefined:

* Variable declare hua hai lekin koi value assign nahi ki gayi.

null:

* Developer manually set karta hai jab woh batana chahta hai ke variable khaali hai.

Example:

let a;
console.log(a);   // undefined

let b = null;
console.log(b);   // null



9. typeof operator 'null' ke liye "object" kyun deta hai?

Answer:
Ye JavaScript ka ek purana bug hai.
typeof null === "object" return karta hai, lekin ye sahi nahi hai.
Backward compatibility ki wajah se isay fix nahi kiya gaya.



10. JavaScript mein array kis type ka data type hai?
